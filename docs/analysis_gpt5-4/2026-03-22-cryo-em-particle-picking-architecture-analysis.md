# SuPicker 面向冷冻电镜颗粒挑选的架构分析

日期：2026-03-22  
分析模型：GPT-5.4

## 1. 分析目标

本文档整理了对当前 SuPicker 项目的系统性分析，目标是从冷冻电镜图像与颗粒挑选任务的真实特点出发，以客观、科学、理性的视角回答以下问题：

1. 当前网络结构是否真正适合冷冻电镜颗粒挑选场景。
2. 现有实现在哪些方面已经合理。
3. 现有实现在哪些关键点上仍然是“通用检测器思路”，而不是“面向 cryo-EM 优化的颗粒挑选系统”。
4. 哪些优化空间是真实存在、且最值得优先投入的。

## 2. 当前实现概况

当前项目的核心架构可以概括为：

- Backbone：ConvNeXt
- Neck：FPN
- Head：CenterNet 风格中心点检测头
- 输入：单通道灰度图
- 输出：热图、尺寸、偏移

对应实现主要位于：

- [supicker/models/detector.py](../../supicker/models/detector.py)
- [supicker/models/backbone/convnext.py](../../supicker/models/backbone/convnext.py)
- [supicker/models/neck/fpn.py](../../supicker/models/neck/fpn.py)
- [supicker/models/head/centernet.py](../../supicker/models/head/centernet.py)
- [supicker/data/target_generator.py](../../supicker/data/target_generator.py)
- [supicker/engine/predictor.py](../../supicker/engine/predictor.py)

它本质上是一个单类、anchor-free 的密集目标检测器，使用中心热图完成颗粒定位，并用 offset 分支补偿输出步长带来的量化误差，同时保留 size 分支来回归颗粒宽高。

## 3. 冷冻电镜颗粒挑选任务的客观特点

要判断结构是否适合，首先必须明确 cryo-EM 颗粒挑选和自然图像检测的差异。

### 3.1 图像层面的特点

- 信噪比极低，背景噪声常常主导视觉外观。
- 颗粒边界弱，轮廓不稳定，不像自然图像目标那样具有清晰语义边缘。
- CTF、defocus、冰厚、污染、碳边等因素会显著影响局部统计分布。
- 同一颗粒在不同微图中的外观变化，往往比自然图像中的“类别内变化”更依赖成像条件而非语义内容。

### 3.2 标注层面的特点

- 颗粒标注通常更接近“中心点监督”，而非高质量目标框监督。
- 同一项目中颗粒直径通常相对集中，粒径是强先验。
- 标注可能存在遗漏、不完备或人为偏差。
- 真正关心的是中心位置、召回率、误检控制，而不是 box 的几何质量。

### 3.3 部署层面的特点

- 推理对象通常是大尺寸 micrograph，而不是固定小图。
- 模型必须面对块边界、一致性拼接、重复检出等工程问题。
- 一个好的 picking 系统不仅要“能检出”，还要尽量减少污染、碳边、聚集体等带来的误检。

## 4. 当前架构适合目标场景的地方

当前架构不是完全不适合。作为 baseline，它有若干合理之处。

### 4.1 CenterNet 路线本身是合理的

当前使用 CenterNet 风格的中心点检测，这比 anchor-based 检测更贴近颗粒挑选任务。

原因：

- 颗粒监督本质上更接近中心点。
- 小目标、密集目标下，anchor matching 往往更脆弱。
- cryo-EM 更关心“哪里有颗粒”，而不是“目标框边界是否精准”。

因此，CenterNet 作为出发点是对的。

### 4.2 单通道输入适配是合理的

主干网络已支持灰度输入，这与微图的输入形态一致，而不是强行沿用 RGB 假设。

### 4.3 旋转、翻转增强方向基本正确

由于颗粒投影方向随机，翻转与旋转增强是合理的基本操作。

### 4.4 随机裁剪是必要的工程策略

大尺寸 micrograph 不可能总以全图形式训练，裁剪训练是必要手段。当前已经具备这一基础能力。

## 5. 当前架构不够适合目标场景的地方

问题不在于“ConvNeXt + FPN + CenterNet”这几个关键词本身，而在于当前实现仍然主要继承了通用检测器的思路，没有充分把 cryo-EM 的物理先验和任务结构写进训练目标与推理逻辑。

### 5.1 没有真正利用粒径先验

这是当前实现最关键的结构性缺口之一。

虽然数据流里保留了 width 和 height 字段，但当前热图高斯半径并没有真正依据粒径变化，而是固定地由 `gaussian_sigma` 产生一个近似常数半径。也就是说：

- 代码表面上保留了尺寸信息。
- 监督核心却没有真正由粒径主导。

而对 cryo-EM 来说，粒径是极强先验，本应直接进入：

- 热图半径
- NMS 半径
- 评估匹配距离
- 推理后处理的空间约束

### 5.2 尺寸分支在当前场景中未必是有效任务

当前 head 同时输出热图、size、offset。但如果训练时大量尺寸只是默认值填充，那么 size 分支学到的很可能只是“近似常数”，而不是与颗粒几何特征有关的稳定模式。

这会带来两个问题：

- 增加优化目标复杂度。
- 把模型容量分配到一个弱监督甚至伪监督子任务上。

对很多 cryo-EM picking 场景而言，中心点比宽高更可靠；如果需要尺度信息，也更适合回归单一粒径标量，而不是自然图像式的 box 宽高。

### 5.3 FPN 成本存在，但没有真正形成多层检测收益

当前结构虽然构建了 FPN，但最终检测只在 P2 上完成，而不是对多个尺度层同时输出检测结果。这意味着：

- 你承担了 FPN 的融合成本。
- 但没有完全发挥典型多尺度检测框架的优势。

如果目标粒径基本稳定，这样做并不是绝对错误；但这也说明当前系统的“多尺度能力”并没有表面上看起来那么强。

### 5.4 当前增强更像自然图像增强，而非 cryo-EM 成像增强

当前默认增强包含 brightness、contrast、gaussian noise 等，这是通用图像任务里常见的增强，但与 cryo-EM 真实难点并不完全对齐。

真正更关键的变化来源于：

- CTF / defocus 风格变化
- 频域响应变化
- 冰厚和局部背景变化
- 污染和局部异常结构
- 低信噪比分布变化

如果增强没有贴近这些因素，那么模型学到的鲁棒性仍然偏“通用图像鲁棒性”，而不是“冷冻电镜鲁棒性”。

### 5.5 后处理参数没有与物理量绑定

当前很多关键阈值是固定像素值，而不是粒径或像素尺寸驱动。

这在 cryo-EM 中并不科学，因为：

- 不同数据集的像素尺寸不同。
- 不同颗粒直径不同。
- 不同缩放和裁剪策略会影响最优空间阈值。

如果这些量都写死，模型的跨数据集迁移性会明显受限。

### 5.6 小 batch 场景下 BatchNorm 可能不稳

cryo-EM 训练中常因图像分辨率高、裁剪大而导致 batch size 偏小。在这种条件下，检测头中的 BatchNorm 很可能成为不稳定因素，尤其当不同微图统计分布差异较大时更明显。

## 6. 综合判断

### 6.1 简洁结论

当前结构适合作为一个可工作的 baseline，但还不能算是一个已经充分贴合 cryo-EM 颗粒挑选目标场景的成熟架构。

### 6.2 更准确的说法

它是一个“合理的第一代实现”，但不是“已经围绕 cryo-EM 特性做过深入任务建模的实现”。

如果场景满足以下条件：

- 单一颗粒类型
- 粒径相对固定
- 标注以中心点为主
- 目标是先获得一个可用 baseline

那么当前架构是合理的。

如果场景是以下类型：

- 多数据集混训
- 粒径变化明显
- 颗粒密集或近邻多
- 微图质量差异大
- 对误检控制有更高要求

那么当前架构还不够，缺口主要不在 backbone 大小，而在任务定义、监督设计、后处理参数化和领域先验建模。

## 7. 真实存在且最值得关注的优化空间

以下优化空间不是“理论上也许可以试”，而是从 cryo-EM 任务本身出发，真正更可能带来收益的方向。

### 7.1 优先把任务重新收缩到中心点检测

优先考虑以下两类方案：

1. 只保留 heatmap + offset。
2. 若确实需要尺度信息，则改为热图 + 偏移 + 单一粒径标量。

理由：

- 当前 size 分支监督不够可靠。
- 颗粒挑选的核心目标是中心位置，而不是框。
- 减少伪任务可以提升训练稳定性。

### 7.2 把所有空间尺度参数统一绑定到粒径

建议统一以下量的来源：

- 热图半径
- 高斯 sigma
- NMS 半径
- 验证匹配距离
- 某些后处理邻域搜索范围

这些量应当由下列信息共同决定：

- 粒子直径
- 像素尺寸
- 模型输出步长

### 7.3 加强困难负样本建模

真实误检通常来自：

- 污染
- 碳边
- 高对比伪峰
- 聚集体
- 背景异常结构

比起“继续堆更大 backbone”，更现实的方向是：

- 增加 hard negative 采样
- 为高误检区域引入额外训练关注
- 在推理阶段结合区域先验抑制伪峰

### 7.4 增强策略改为物理退化驱动

相比普通 brightness / contrast 扰动，更值得做的是：

- defocus 风格变化
- 频域模糊与带宽变化
- 更真实的低信噪比噪声模型
- 局部背景梯度变化
- CTF 风格扰动

### 7.5 重新审视 ConvNeXt + FPN 的性价比

ConvNeXt 来自自然图像预训练分布，虽然可以用，但对 cryo-EM 这种低语义、低对比、强物理先验任务，不一定是最佳性价比选择。

更值得做的不是先默认“大 backbone 更强”，而是做严谨消融：

- ConvNeXt + FPN + P2 检测
- ConvNeXt 直接单层高分辨率检测
- 更轻 backbone
- U-Net 风格结构
- 高分辨率保持型结构

### 7.6 完善大图滑窗推理与边界一致性

大尺寸 micrograph 推理不能只停留在“模型能输出颗粒坐标”，还需要系统处理：

- 滑窗重叠
- 边界重复检出
- 块间一致性
- 边缘漏检

这类工程能力对真实部署很重要。

### 7.7 优化归一化和特征统计稳定性

可以重点评估：

- GroupNorm 替代 BatchNorm
- 更稳定的图像标准化方式
- 局部标准化或适度频域预处理

## 8. Top 10 最值得优先落地的改动点

以下按“预期收益高到低”排序，并结合工程风险给出建议。

### 1. 去掉或降级尺寸分支，优先收缩到中心点检测

原因：当前尺寸监督较弱，size 分支可能主要在学习默认值，而不是有效几何信息。

建议：

1. 先做 heatmap + offset 版本。
2. 再做 heatmap + offset + diameter 版本对照。

预期收益：高  
实施风险：低到中

### 2. 把热图半径改成粒径驱动，而不是固定 sigma

原因：这是当前监督定义与 cryo-EM 任务几何结构最直接的错位。

预期收益：高  
实施风险：低

### 3. 把 NMS 半径和验证匹配距离改成粒径驱动

原因：固定像素阈值不具备跨数据集和跨粒径的一致性。

预期收益：高  
实施风险：低

### 4. 把头部 BatchNorm 替换为 GroupNorm

原因：更适合小 batch、高分布波动场景。

预期收益：高  
实施风险：低

### 5. 用更贴近 cryo-EM 的增强替代自然图像式增强

原因：领域泛化能力更依赖物理退化建模，而不是普通亮度对比度扰动。

预期收益：高  
实施风险：中

### 6. 把纯随机裁剪改成混合采样策略

建议将以下 patch 混合采样：

1. 以颗粒为中心的正样本 crop
2. 困难负样本 crop
3. 少量纯随机背景 crop

预期收益：中到高  
实施风险：中

### 7. 引入 hard negative 机制

原因：真实误检多来自背景伪峰而不是类别混淆。

预期收益：中到高  
实施风险：中

### 8. 重新评估 FPN 的必要性

原因：当前检测只在 P2 完成，FPN 的额外价值需要用消融实验而不是直觉来证明。

预期收益：中  
实施风险：中

### 9. 让输出步长和早期下采样变成可配置实验项

原因：极小颗粒、密集颗粒可能对输出分辨率更敏感。

预期收益：中  
实施风险：中到高

### 10. 建立自动阈值扫描流程

原因：固定 score threshold 不具备稳健性，评估流程应自动寻找最优 operating point。

预期收益：中  
实施风险：低

## 9. 分阶段重构路线图

建议按风险递增、收益递进的方式实施，而不是一次性大改架构。

### 阶段一：低风险，高收益

这一阶段尽量不动整体骨架，只修正当前最明显的不贴合点。

目标：

1. 修正任务定义与监督定义。
2. 让关键空间参数与粒径一致。
3. 提升训练稳定性与评估可靠性。

建议内容：

1. 去掉或弱化尺寸分支。
2. 把热图半径改为粒径驱动。
3. 把 NMS 半径和评估匹配距离改为粒径驱动。
4. 把头部 BatchNorm 改成 GroupNorm。
5. 增加自动阈值扫描流程。

这一阶段不是在“赌更强模型”，而是在修正当前实现与目标场景之间最直接的错位。

### 阶段二：中风险，中高收益

这一阶段开始改训练分布和领域增强，但尽量不推翻主干结构。

目标：

1. 让模型看到更合理的数据分布。
2. 让增强更贴近 cryo-EM 真实退化。
3. 更系统地控制误检来源。

建议内容：

1. 把纯随机裁剪改成混合采样策略。
2. 引入更贴近 cryo-EM 的增强：噪声、频域扰动、defocus / CTF 风格变化等。
3. 建立 hard negative 机制。
4. 优化图像标准化方式。

### 阶段三：高风险，高成本，潜在高收益

这一阶段才建议动更深的结构问题。

目标：

1. 重新评估 backbone / neck 的性价比。
2. 评估更高分辨率输出的必要性。
3. 必要时评估从检测范式转向 dense response 或弱分割范式。

建议内容：

1. 消融 FPN 是否必要。
2. 让输出步长与 stem 下采样变为可配置实验项。
3. 评估 CenterNet 之外的范式，例如响应图或分割式 picking。
4. 有条件时考虑 cryo-EM 域内自监督预训练。

## 10. 最务实的首轮开发包

如果准备立刻开始代码改造，建议第一轮只做以下 4 项：

1. 去掉或弱化尺寸分支。
2. 热图半径、NMS 半径、匹配距离全部绑定粒径。
3. 头部 BatchNorm 改成 GroupNorm。
4. 增加自动阈值扫描。

原因：

- 风险低
- 改动逻辑闭合
- 容易做回归验证
- 更可能带来真实收益，而不是表面复杂度提升

## 11. 最终结论

当前 SuPicker 架构不是完全不适合冷冻电镜颗粒挑选，但它更像一个“合理的通用检测 baseline”，而不是一个已经深度吸收 cryo-EM 任务先验的成熟方案。

真正最值得优先优化的，不是继续堆更大的 backbone，也不是先追求更复杂的 neck，而是：

1. 把任务定义重新收缩到颗粒中心定位本质。
2. 把粒径、像素尺寸等物理先验写入监督和后处理。
3. 把增强、采样和误检控制机制从“通用图像逻辑”转向“冷冻电镜逻辑”。

如果这些基础层没有做好，那么即使更换更大的网络，收益也很可能低于预期。
