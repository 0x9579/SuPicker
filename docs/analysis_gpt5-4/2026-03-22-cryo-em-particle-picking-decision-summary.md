# SuPicker 架构评估决策摘要

日期：2026-03-22  
分析模型：GPT-5.4

## 一句话结论

当前 SuPicker 架构适合作为冷冻电镜颗粒挑选的可用 baseline，但还不能算是一个已经充分贴合 cryo-EM 目标场景的成熟方案。

问题的核心不在于 backbone 还不够大，而在于当前实现仍然主要继承了通用自然图像检测器的思路，没有把 cryo-EM 中最关键的粒径先验、低信噪比成像特性、困难负样本和后处理物理尺度约束真正写进系统。

## 当前架构的合理之处

1. 使用 CenterNet 风格中心点检测而非锚框检测，这与颗粒挑选更接近点监督的事实是匹配的。
2. 支持单通道输入，符合 cryo-EM 微图的灰度图像形态。
3. 包含旋转、翻转、裁剪等基础数据增强，工程上具备可训练性。
4. 已具备完整的训练、推理、目标生成和后处理链路，适合用作第一代基线系统。

## 当前架构的关键不足

1. 粒径先验没有真正进入监督核心。热图半径、NMS 半径、评估匹配距离都没有系统地绑定到粒径与像素尺寸。
2. 尺寸分支未必真正有效。若大量样本尺寸来自默认值填充，那么 size 分支更可能在学习常数，而非有效几何信息。
3. FPN 存在计算成本，但检测实际只在单层输出上完成，多尺度收益并不充分。
4. 当前增强更偏自然图像逻辑，而不是 cryo-EM 真实成像退化逻辑。
5. 对污染、碳边、聚集体、高对比伪峰等真实误检来源缺少显式 hard negative 机制。
6. 小 batch 训练下，检测头中的 BatchNorm 可能不稳定。

## 对项目现阶段的总体判断

如果项目目标是：

- 在单一颗粒类型数据上快速建立可用 baseline
- 用较标准的深度学习检测范式完成第一版系统
- 先验证端到端流程是否成立

那么当前架构是合理的。

如果项目目标是：

- 面向更真实、更复杂的 cryo-EM 数据分布
- 获得更稳定的跨数据集泛化
- 降低误检并减少人工筛选负担
- 在不同粒径、不同像素尺寸和不同微图质量条件下保持一致表现

那么当前结构仍有明显的真实优化空间。

## 最值得优先做的 4 项改动

### 1. 弱化或移除尺寸分支

优先把任务收缩到“中心热图 + 偏移”，必要时再把宽高回归改成单一直径标量。

原因：颗粒挑选的核心是中心定位，不是 box 回归；弱尺寸监督会增加训练噪声。

### 2. 把所有空间半径统一改为粒径驱动

至少统一以下量：

- 热图高斯半径
- NMS 半径
- 验证匹配距离

原因：这一步最直接决定监督几何结构是否符合 cryo-EM 任务本身。

### 3. 将检测头 BatchNorm 改为 GroupNorm

原因：更适合 cryo-EM 常见的小 batch、高分布波动训练条件。

### 4. 建立自动阈值扫描与验证流程

原因：固定 score threshold 和固定半径不具备跨数据集稳健性，应通过验证集自动搜索最优 operating point。

## 中期优化方向

在完成上述低风险修正后，建议进入第二阶段优化：

1. 把随机裁剪改成正样本、困难负样本、背景 patch 的混合采样。
2. 用更贴近 cryo-EM 的增强替代普通亮度对比度增强。
3. 建立 hard negative 机制，压制污染和背景伪峰误检。
4. 重新评估图像标准化和局部预处理方式。

## 高风险但值得后续验证的方向

1. 重新评估 FPN 是否必要。
2. 评估更高分辨率输出步长是否更适合小颗粒或密集颗粒。
3. 视实验结果决定是否从 CenterNet 进一步转向响应图或分割式 picking 范式。
4. 若数据规模允许，考虑 cryo-EM 域内自监督预训练。

## 最终建议

当前最应该做的不是盲目换更大的 backbone，而是先把以下三件事做对：

1. 让任务定义回到颗粒中心定位本质。
2. 让粒径和像素尺寸先验进入监督与后处理核心。
3. 让训练分布和增强策略更接近 cryo-EM 的真实成像条件。

只有在这些基础层已经被修正之后，再去比较更复杂的 backbone、neck 或检测范式，结论才更可信。
