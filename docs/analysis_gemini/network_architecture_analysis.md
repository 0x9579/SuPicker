# SuPicker 网络结构适用性与优化空间分析

本报告基于冷冻电镜（Cryo-EM）图像本身的独特物理、信号特性以及目标颗粒的表现形式，对 `SuPicker` 当前采用的 `ConvNeXt + FPN + CenterNet` 网络结构进行客观、科学、理性的分析，并指出真实的优化空间。

## 一、 冷冻电镜图像和颗粒的物理与数据特点

在分析网络结构之前，必须明确目标场景的数据特点（Domain Gap），这是所有优选架构的基础：

1.  **极度恶劣的信噪比 (Extremely Low SNR):** 电子束剂量受限，导致图像被散粒噪声、结构噪声（碳膜、冰层污染）淹没。颗粒（纯相位物体）的对比度极低，很多时候人眼甚至无法直接辨认。
2.  **高频信息的震荡与丢失 (CTF 调制):** 由于散焦（Defocus）成像，衬度传递函数（CTF）导致图像呈现明显的 Thon 环干涉（低频有衬度，高频有相位反转）。这也导致所谓的“边缘”和普通图像的“边缘”是完全不同的物理概念。
3.  **目标尺度高度一致性 (Scale Consistency):** 最大的区别在于——**在某一批特定的冷冻电镜数据集中，特定的生物大分子（如某种蛋白质复合物）的物理尺寸是基本恒定的**（比如都是 15nm 左右的核糖体）。只会存在不同取向（Orientation）的 2D 投影差异，而极少存在自然图像中（如一辆车在近处和远处的视觉遮挡）几十上百倍的尺度缩放现象。
4.  **正样本高度稀疏且存在海量“假阴性”标注:** 图像尺寸极大（如 $4K \times 4K$），有效颗粒在全局占比极小，且背景常常存在极具迷惑性的“假颗粒”结构。同时，人工标注普遍存在大量漏标（Unlabeled Positives / False Negatives），网络在训练时如何容忍而不是错误地去惩罚这些未标注的隐藏真样本，是成功与否的关键。

## 二、 当前网络结构 (`ConvNeXt + FPN + CenterNet`) 适用性剖析

目前这套结构是一个非常标准、高效的通用目标检测范式（常用于 COCO 数据集等自然场景）。但在冷冻电镜场景下，它的表现和设计初衷存在以下错位与隐患：

### 1. ConvNeXt (Backbone)：强有力的特征提取，但存在极端分辨率流失风险
*   **适用性：** ConvNeXt 引入了 Depthwise Convolutions 和更大的有效感受野，能够很好地提炼深层语义和空间特征，是一个极优秀的 Backbone。
*   **不适痛点（Stem层的强制降采样）：** 目前代码 `convnext.py` 中的 `Stem` 层直接使用了 `kernel_size=4, stride=4`。这使得输入图像在一开始就直接去掉了 75% 的空间分辨率（图像尺寸变为十六分之一）。在 Cryo-EM 极低信噪比的环境下，某些小质量的蛋白复合物（如 <100 kDa）原本只占据十几个像素大小，经过一次 $4\times 4$ 下采样后就会面临高频边缘信息和定位信息的严重流失，甚至直接与背景噪声混合，大幅降低了网络对极微小结构特征的捕捉极限。

### 2. FPN (Feature Pyramid Network, Neck)：多层级降配，功能产生冗余
*   **适用性：** FPN 存在的意义是**特征金字塔多尺度融合**，初衷是为了让模型（例如 Faster R-CNN）既能检测超大目标又能检测超小目标。
*   **不适痛点（多余的尺度开销）：** 在特定的冷冻电镜处理流中，单个项目里的蛋白质大小几乎是恒定的。让网络强制在不同尺度（P2, P3, P4）上进行多级特征提取往往是冗余的，既增加了前向推理和训练计算开销，又因为网络必须在深层降频特征和浅层高频特征间平衡而可能导致目标位置发生次像素偏移。此外，目前仅在 FPN 最高的保留分辨率 P2 ($1/4$ scale) 上做头部的检测，依然先天性地损失了高定位精度要求。

### 3. CenterNet (Head)：核心思想完美契合，但存在冗余的回归分支
*   **适用性（完美契合点）：** CenterNet 摒弃了 Anchor 和繁琐的 Non-Maximum Suppression (NMS)。它的核心是基于热力图（Heatmap）的点估计预测，这一点**非常完美地契合了冷冻颗粒挑选任务**（挑选只要求目标的“几何中心二维坐标”，并不需要精细的边界框）。
*   **不适痛点（多余的 `Size` 与 `Offset` 回归分支）：** 目前 `supicker/models/head/centernet.py` 中，网络保留了 `Size`（宽/高）预测和子像素 `Offset` 回归预测。既然目标（蛋白）尺寸具备极强的固定先验，为了在对比度极其微弱且边界模糊的电镜图中去强行回归出一个“包围框的长和宽”，会引入非常多无谓的噪声梯度（Noise Gradients）。这些噪声梯度在反向传播时，反而会干扰或拖累核心的 `Heatmap` 学习性能。让网络在噪声中“盲估”颗粒边界，不如直接在后处理阶段使用给定的超参数（Box Size）截取。

## 三、 基于上述分析的真实优化空间 (Optimization Space)

综合来看，要在当前的 SuPicker 架构上做真正能带来冷冻电镜领域（尤其是低SNR和小颗粒）实质性提升的优化，可落地的改造方向如下：

### 🚀 优化方向 1：大刀阔斧地简化 Head，移除 Size 预测（降噪提效）
既然目标尺寸在单一任务中是先验常数，我们完全不需要网络在微弱的信噪比下艰难地去估计边界框大小。
*   **具体实施：** 砍掉 `Size` 回归分支（甚至可选地移除 `Offset` 分辨支）。让模型所有的拟合能力（Capacity）和梯度更新，专注在**输出一张高质量的、关于潜在颗粒中心点分布的高斯概率 Heatmap**。后处理时，颗粒的范围框直接以超参数传入一个指定的直径大小生成。这样可极大降低网络在模糊边缘的过拟合风险和训练震荡。

### 🚀 优化方向 2：针对小蛋白重塑 Backbone 的 Stem 分辨率（信息保真）
目前的 $4\times4\ (stride=4)$ 下采样起步，对于小于 100kDa 或直径在几纳米级别的小蛋白过于残忍。
*   **具体实施：** 修改 `ConvNeXt` 的 `Stem` 模块参数配置，降低下采样的步伐。例如改可将初始感受野保留在 `stride=2`，或者直接重构骨干网络采用形如高分辨率网络（HRNet）那样的架构，始终维持一个与原始图像同级别的超高分辨率主干以无损捕获微弱的高频衍射信息特征。

### 🚀 优化方向 3：解决标注“假阴性”硬伤，引入 PU-learning
面对 Cryo-EM 的极低分辨图象，人工标注的数据必然面临**大量漏标（假阴性，False Negatives）**。这种情况下如果仍使用标准的 Focal Loss 去强硬地把所有背景（包括漏标的真实颗粒）置 0 进行打压，网络就是在错误地逼迫自己放弃一部分正确的判断。
*   **具体实施：** 在 Heatmap 概率预测的损失函数上引入 **PU Learning (Positive-Unlabeled)** 或基于网络自信度的 **Ignored Regions (Confidence Masking)** 技术。在反向传播计算 Loss 时，判断如果有无标签的区域网络仍独立预测出了极高置信度，这极大概率是真实颗粒。此时应阻断该位置的负样本梯度惩罚（Gradient Masking），不对其施加负向打压。

### 🚀 优化方向 4：跨越图像领域的自监督预训练 (解决 Domain Gap)
目前的框架采用了 `torchvision` 跑过 ImageNet（自然 RGB 图像，有着清晰色彩与轮廓）的预训练参数，且靠一层像素均值粗暴地降维成了单通道供冷冻图（严重噪声 + 灰度 + 干涉条纹）使用。虽然网络最终可以凑合收敛，但预训练的感受野和特征偏好并不是为解决散粒噪声而生的。
*   **具体实施：** 采用公共领域巨大的冷冻电镜图像数据集（例如 EMPIAR 上多达数百TB量的源 Micrographs），抛弃标签，进行 **自监督学习（Self-Supervised Learning，如 Masked Autoencoder (MAE) 重构策略）** 作为骨干的预训练基底。这将使 ConvNeXt 引擎直接获得理解物理冰层噪声特征的基础鉴别力，带来降维打击式的提取性能提升。
